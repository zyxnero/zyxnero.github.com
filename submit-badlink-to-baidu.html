制作提交百度死链

Oct 06,2014

Update

  • 404.bropaul.com 已失效，将 404 页 CNAME 到 github 的 404 页，现百度不良收录亦减少至 0。

去年开始折腾 Pelican 之后，很多博文就被我遗弃了。加之 DNSPod 的账号被盗，入侵者添加的泛域名规则导致 Google 与 Baidu 都收录了很多很多（3 位数）的恶意解析链接，删除泛域名规则之后导致恶意解析链接成为死链。

后将泛域名解析至 404.bropaul.com，Google 的不良收录很快恢复至正常，然而 Baidu 的收录却很难恢复。收藏夹中的《如何制作提交百度死链》也已挂掉，所以在此留一份存根。

提交死链方法

前往百度站长平台：

制作死链文件，Baidu 支持两种格式，txt 和 xml。这里推荐 xml，格式跟 sitemap 一样。可以参考本人现有的 死链文件，经测试可用。

上传 xml 文档，设置好更新周期。至此，需要做的就是耐心等待百度处理了。
