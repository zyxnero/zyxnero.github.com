更改域名解析

前阵子注册了个 QQ 企业邮箱，结果经常显示“MX 记录与 CNAME 记录冲突”，一直没弄明白是哪发生了问题。于是科普了一下 CNAME 的含义1。

Google 的说明

正規名稱記錄又稱 CNAME 紀錄，是一種 DNS 紀錄類型，可將別名對應至實際或「正規」的網域名稱。CNAME 紀錄的用途通常是將子網域 (例如：「www」或「mail」) 對應至代管子網域內容的網域。舉例來說，CNAME 紀錄可以將網址「www.example.com」對應至「example.com」網域的實際網站。

Github 的说明

Warning: You should not use a CNAME record with an apex domain, as this may cause issues with other services on the apex domain, like email.

上面的引用表达的意思：根域名最好不要做 CNAME，否则会造成邮件解析出错（除非邮件解析与 www 解析的目的地是同一服务器）。

正确的解析方式

在域名运营商处修改（假设要把 www 与根域名指向同一个站点）：

CNAME：www.example.com => username.github.io
A：example.com => 192.30.252.153
