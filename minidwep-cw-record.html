RTL8187 CW 小记

Nov 29,2014

搬“家”之后就得 CW 以解燃眉之急。我用的是傻瓜工具 VMware + CDLinux + minidwep。

VMware 提示 USB 设备正在使用中

我用的是绿色版的 VMware Workstation 8，所以出现这个提示的时候我的第一反应是 VMware 的 USB 服务没有启动。事实是 VMware Workstation 8 不支持 USB3.0。根据电脑现有的情况（只有一个 USB2.0 接口）我选择了升级 VMware 到 10.0.3。

保存 minidwep 进度

跑 pin 也是要有经验的，pin 排序合理了能省不少时间。我是个外码，所以需要的时间比较长。CDLinux 太过小巧，有必要进行进度保存。

  1. 记住排序码；

  2. 到 /tmp/minidwep 目录下打开需要保存进度的 mac 地址对应的 .wpc 文件，记下第一行数字；

同上，恢复进度为其逆向过程：

  1. 使用 minidwep 扫描并选中需要恢复进度的信号；

  2. 输入之前保存的排序码；

  3. 到 /tmp/minidwep 目录下打开需要恢复进度的 mac 地址对应的 .wpc 文件，输入上次记录的第一行数字；

  4. 点击 Reaver 接着 pin。

PS：如果通过某种算法已经能够确定对方的 pin 码，则通过修改 /tmp/minidwep 目录下的对应的 .wpc 文件中的前 3 行数值就能直接定位到 pin 码。第一行数值与 pin 码的前 4 位关联（故最大修改为 4 位数）；第二行数值与 pin 码后 4 位中的前 3 位关联（故最大修改为 3 位数）；第三行数值与 pin 码的最后一位（校验位）关联，故为 1 位数。假定某 pin 码为 35108146，pin 排序为 0123456789，则将对应的 .wpc 文件中的前 3 行数值分别修改为 3510，814，6 即可直接定位。
